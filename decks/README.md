# Veroax decks

Two PowerPoint files for outbound conversations:

- `veroax-vc-pitch.pptx` (10 slides). For VC, angel, and other
  investor conversations. Cover, problem, solution, how it works,
  market, business model, why now, defensibility, roadmap + use
  of funds, ask.
- `veroax-brokerage-sales.pptx` (12 slides). For online sales
  conversations with brokerage leadership. Cover, the agent's
  reality, hidden cost, what Veroax does, inside the report,
  quality safeguards, white-label brand experience, plans, ROI
  math (15-agent worked example, 14x return), privacy and
  compliance, two-week implementation timeline, next step.

## How they're built

Both decks are generated by Node scripts using
[pptxgenjs](https://github.com/gitbrent/PptxGenJS). The shared
brand tokens (colors, fonts) live in `_brand.js` so both files
stay visually consistent with the rest of the marketing site
(coral wordmark, teal accent, navy depth on title and closing
slides).

```
decks/
├── _brand.js                          shared color + font tokens
├── vc-pitch.js                        VC pitch deck source
├── brokerage-sales.js                 brokerage sales deck source
├── veroax-vc-pitch.pptx               generated output (10 slides)
├── veroax-brokerage-sales.pptx        generated output (12 slides)
└── package.json                       pptxgenjs dependency
```

## Regenerating

Edit either `vc-pitch.js` or `brokerage-sales.js` and re-run:

```bash
cd decks
node vc-pitch.js          # → veroax-vc-pitch.pptx
node brokerage-sales.js   # → veroax-brokerage-sales.pptx
```

Both scripts overwrite the existing `.pptx` files in place.

## Editing in PowerPoint / Keynote

The generated files open natively in PowerPoint (Mac and Windows),
Keynote, and Google Slides. If you make manual edits in the GUI
and want to keep them, save the file under a new name so the
next regeneration does not overwrite your work.

## Adjusting numbers

A few numbers in the decks are placeholders meant for you to
verify or replace:

- VC deck, slide 5 (Market):
  - "350K+ licensed real-estate agents in California"
  - "400K+ annual residential transactions"
  - "$1.4B addressable spend"
  These are reasonable practitioner estimates. Replace with
  freshly-pulled CAR / NAR numbers before you present.

- VC deck, slide 9 (Use of funds):
  - The 40 / 30 / 15 / 15 split assumes a seed-stage raise. If
    you're raising a different size or stage, edit the percentages
    in `vc-pitch.js` and re-run.

- Brokerage deck, slide 3 (Hidden cost):
  - "$300+ paralegal-equivalent labor"
  - "1 in 6 deals hit a renegotiation"
  - "$10K+ average concession"
  These are credible practitioner ranges. Tighten with real data
  if you have brokerage-specific numbers from a pilot.

- Brokerage deck, slide 9 (ROI math):
  - The 15-agent / 2-deal / $75 hourly worked example is meant to
    be tweakable in front of the prospect. If you're meeting with
    a 40-agent office, edit the agents number and reshow.

## What's deliberately NOT in either deck

- No financial projections (we don't have the data to defend them
  yet, and made-up TAM-times-take-rate math reads as a red flag
  to sophisticated investors).
- No customer logos (we have not closed a brokerage yet, and a
  page of "soon" logos hurts more than it helps).
- No product screenshots (the sample PDF cover on brokerage
  slide 7 is a mock that conveys the look without committing to
  a specific page that may change).

Add any of these once you have the underlying material.
